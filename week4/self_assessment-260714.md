## ① สรุปเนื้อหาที่เรียน — ฉันได้เรียนรู้อะไรในสัปดาห์นี้?

**Asset Creation For Games**

**1.Data Import and Export** : กระบวนการจัดการข้อมูลการนำ Asset มาใช้ในเกมประกอบด้วย 2 ขั้นตอนหลัก

    -Export: ดึงข้อมูลออกจากโปรแกรมวาดหรือปั้น(Art package) ได้ออกมาเป็น ไฟล์ตัวกลาง(Intermediate file)

    -Import: การนำไฟล์ตัวกลางนั้นมา แปลงและ Optimize ให้เข้ากับระบบของ Game Engine เพื่อพร้อมนำไปใช้ในเกม

**2.Artist Assets Creator**

    **1) 2D Animation**

    -Pixelorama: แอปทำ Pixel art แบบ Open-source ที่พัฒนาด้วย Godot engine

    -Universal Sprite Sheet Character Creator: เครื่องมือช่วยสร้าง Sprite sheet สำหรับตัวละครแอนิเมชัน 2D แบบหลายมุมมอง

    -Spine: ใช้ทำแอนิเมชัน 2D โดยการสร้าง Sprite sheet หรือใช้เทคนิคอย่าง Onion skinning และ Rotoscoping

    -TexturePacker: ใช้สร้างและจัดการSprite Sheets หรือ Texture Atlas โดยจะนำภาพกราฟิกขนาดเล็กหลายๆ ภาพมารวมไว้ในไฟล์รูปภาพขนาดใหญ่ไฟล์เดียวอย่างเป็นระเบียบ

    **2) Effects and FX**

    -Effect: ผลลัพธ์ทางตัวเลขหรือกลไกเกม เช่น ถ้าตัวละครโดนลูกไฟ แล้วเลือดลดลง ค่าพลังชีวิตที่ลดลงนั้น

    -FX: เอฟเฟกต์ทางภาพที่เห็นบนจอ เช่น particle ระเบิดตูมตามของลูกไฟ

    **3) Creating Run-Time 3D Animation** : ใช้ Game Engine ในการขยับโครงร่างโมเดล(Skinned Mesh) และ render ออกมาแบบ real time

    -ความพิเศษ: Animation นี้ต้องการข้อมูลเพิ่มเติม เพื่อใช้ส่งสัญญาณ(Trigger) สั่งให้ Particle effects เสียง ระบบ AI หรือเหตุการณ์ใน gameplay ทำงานร่วมกันได้อย่างถูกต้องตามจังหวะ

**3.Optimizing Assets**

    -Optimizing Assets 2D: ปัญหาส่วนใหญ่เกิดจาก effect สร้าง particle มากเกินไป และการ render พื้นหลังโปร่งใส (Overdraw) ทำให้ FPS ตก แก้ไขได้โดยการรวม Textures ย่อยๆ เข้าเป็น **Texture Atlas** ผืนเดียว เพื่อลดจำนวน Draw calls ให้ GPU ทำงานง่ายขึ้น

    -Optimizing Assets 3D: การ Swap เปลี่ยนโมเดลความละเอียดสูงเป็นความละเอียดต่ำลงเมื่อวัตถุอยู่ไกลจากกล้อง เรียกว่าระบบLOD (Level of Detail)

**4.Game Design Assets Creator**

    **1) Prototype Tools**

    -Roblox: เป็น platform online ไม่ใช่ตัวเกมเดี่ยวๆ แต่เป็น platform สำหรับเล่นเกมที่สร้างขึ้นโดยนักพัฒนาคนอื่นๆที่มีออกมาหลากหลายแมพ

    -Unity  Microgame เช่น LEGO® Microgame เป็นเครื่องมือช่วยเริ่มต้นศึกษาการใช้งาน Unity ผ่านการทำมินิเกมเลโก้

    **2) บทบาทหน้าที่ในทีมออกแบบ**

    -System Designers: ทำหน้าที่พัฒนาและออกแบบระบบ Gameplay mechanics เช่น การกำหนดว่าผู้เล่นจะกระโดดอย่างไร

    -Level Designers: ทำหน้าที่กำหนดและวางตำแหน่งของ Art assets ต่างๆภายในฉาก ออกแบบฉากความยากที่เหมาะสม

    **3) Types of Level Editor:** Level Designer จะทำงานผ่านเครื่องมือสร้างฉาก 3 ประเภทหลักๆ

    -Tile-Based Editors: นำ Tiles เดิมมาคัดลอกและวางซ้ำๆ เพื่อสร้างฉากขนาดใหญ่ได้อย่างรวดเร็ว ใช้ได้ทั้ง 2D 3D

    -CSG Editors (Constructive Solid Geometry Editors): เป็น Tool ที่มาจากเกม 3D ยุคแรกอย่าง*Quake* และ *Unreal* เป็น Tool หรือระบบแก้ไขรูปทรงเรขาคณิต 3D

    -Terrain Editors: เป็น Tool สำคัญสำหรับสร้างและปรับแต่งสภาพแวดล้อม 3 มิติ เช่น ภูเขา หุบเขา ทะเลทราย และแหล่งน้ำให้สมจริงจาก Height map

## ② สรุปการฝึกปฏิบัติ — ฉันได้ทดลองและฝึกเขียนโปรแกรมอะไรบ้าง?

ฝึกการเขียน flowchart ของ Machine ของเกมเที่ออกแบบ ฝึกการคิดการวางแผนของเกมว่าต้องใช้ assets อะไรบ้าง ทำ asset-list-pipeline-flows ใน Vscode ฝึกการหา asset ที่ใช้ได้จาก website ต่างๆ ที่ได้มีการกำหนดชัดเจนว่าสามารถนำไปใช้ทำอะไรได้บ้าง เรียนรู้เกี่ยวกับเรื่องลิขสิทธิ์ของผลงานของเจ้าของ assets เช่น itch.io, unity asset store พวกเสียงต่างๆจาก freesound และการให้เครดิตแก่เจ้าของผลงานอย่างถูกต้อง
