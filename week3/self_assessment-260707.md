## ① สรุปเนื้อหาที่เรียน — ฉันได้เรียนรู้อะไรในสัปดาห์นี้?

**1. 4 Parts of Basic Game Element**

    **-Mechanics** (กลไกเกม): กฎกติกาและขั้นตอนในการดำเนินเกม

    -**Story** : ลำดับเหตุการณ์ที่เกิดขึ้นในเกม เนื้อเรื่องในเกม แบ่งเป็นสเปกตรัมตั้งแต่เนื้อเรื่องเส้นตรงแบบ Traditional Stories ไปจนถึงเนื้อเรื่องที่ผู้เล่นสร้างขึ้นเองทั้งหมด

    **-Aesthetics** (สุนทรียศาสตร์): ภาพ, เสียง, และความรู้สึกที่ผู้เล่นได้รับจากเกม มี 8 Kinds of Fun (SENSATION, FELLOWSHIP, CHALLENGE, FANTASY, NARRATIVEDISCOVERY, EXPRESSION, SUBMISSION)

    **-Technology** : วัสดุหรือระบบที่ทำให้เกมเกิดขึ้นได้ แบ่งเป็นระดับโครงสร้างพื้นฐาน เช่น การเรนเดอร์กราฟิก และระดับตกแต่ง เช่น แสงเงา/เอฟเฟกต์

**2.Game Design Document: GDD*
*-*Game Pitch Document:**เอกสารนำเสนอไอเดียเกมในระยะเวลาสั้น ๆ (Elevator Pitch ประมาณ 2 นาที) เพื่อดึงดูดผู้ร่วมทุนหรือผู้จัดจำหน่าย*

**
    -Tiny GDD**: รูปแบบการสรุปไอเดียเกมสั้น ๆ ลงในกระดาษแผ่นเดียวที่พับเป็นเล่มเล็กๆ ได้ ประกอบด้วย:  Core Mechanic, Main Character / Objects / Obstacles, Level Design & Act Now: Prototype

**3.ประเภทของผู้เล่น (Bartle Player Types)
	-Socializers:** ชอบปฏิสัมพันธ์กับผู้เล่นคนอื่น  

**
    -Explorers:** ชอบสำรวจและปฏิสัมพันธ์กับโลกในเกม

**
    -Achievers:** ชอบทำภารกิจ ล่าความสำเร็จ และส่งผลต่อโลกในเกม

**
    -Killers:** ชอบแข่งขันและมีแอ็กชันกับผู้เล่นคนอื่น

## ② สรุปการฝึกปฏิบัติ — ฉันได้ทดลองและฝึกเขียนโปรแกรมอะไรบ้าง?

ฝึกการใช้ git ในการทำงานร่วมมือกับเพื่อน เวลาต้องการ push งานส่งในไฟล์งาน ฝึกการ push, commit, pull ลงเครื่องตัวเองและการเชื่อม git กับเพื่อนในกลุ่มใน visual studio code การวางแผนงานของ game core game หลัก Mechanics ในเกมการกำหนดแนวเกม GDD
