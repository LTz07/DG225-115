## ① สรุปเนื้อหาที่เรียน — ฉันได้เรียนรู้อะไรในสัปดาห์นี้?

**The Basic Functional Of A Pipeline**

**1.What Pipelines Do?**

-บริหารจัดการการไหลของข้อมู จากกระบวนการหนึ่งไปอีกกระบวนการหนึ่ง

-กำกับการทำงานจากงานหนึ่งไปยังอีกงานหนึ่ง

**2.Defining Your Goals**

-Core Values: ระบุ Core Game หลัก ที่จำเป็นต้องทำให้เล่นได่หรือใช้งานได้ก่อนฟีเจอร์อื่น

-ขนาดและข้อจำกัดของ Project: ขนาดของความซับซ้อน, การนำเครื่องมือเก่ากลับมาใช้ใหม่, ข้อจำกัดทางเทคนิค และเงื่อนไขด้านเวลา

-ปัจจัยด้านทีมงานของ Artists: ความถนัดของ Artist ต่อ software, การให้ Feedback ตามประสบการณ์, จำนวนและตำแหน่งของทีมงานรวมถึงค่านิยมหลักของบริษัท

**3.Defining Standards**

-Technical Standards: กำหนดประเภท software ที่จะใช้, format file สำหรับส่งต่อข้อมูล, ขั้นตอน Workflow, คุณลักษณะ Metadata และมาตรฐานการจัดตารางงานการตรวจงาน

-Micro Pipelines: สเกลย่อยของ Pipeline ที่กำหนดกฎเฉพาะส่วน แต่ก็มีความยืดหยุ่นพอที่จะยอมรับข้อยกเว้นในสถานการณ์จริงได้

**4.Strategies for Managing Data**

-Directory Structure: จัดระเบียบโครงสร้างโฟลเดอร์ตามหน้าที่ของผู้ใช้งาน เพื่อแยกไฟล์แต่ละประเภทออกจากกันชัดเจน เช่น แยก source code ออกจากไฟล์ Art

-File-Naming Conventions: ตั้งชื่อไฟล์ให้เป็นระบบเพื่อให้เป็น Identifier ที่ไม่ซ้ำกัน เช่น XX_TR_001 (XX = โปรเจกต์, TR = ฉาก, 001 = ช็อต)

-Metadata & Tagging: กำหนดข้อมูลอธิบายไฟล์ เช่น ประเภท, ชื่อ, วันที่ และติด Tag เพื่อเพิ่มประสิทธิภาพในการค้นหามากกว่าโครงสร้างโฟลเดอร์แบบเดิม

-Asset Browser: tools ค้นหาและ Listing Assets ในระบบ มีฟังก์ชันหลัก 4 ด้าน: Search, ดูความเชื่อมโยง(Dependencies), Browsing และ Tagging
